# 聊天页面设计对话归档摘要

> 本文件原先保存了一份 2300 多行的英文设计工具对话。其内容包含大量临时工具日志、重复尝试和已经过时的实现细节。为保持仓库文档中文化并降低维护噪音，现整理为以下设计摘要。当前实现以代码和 [UI 设计规范](../../UI-DESIGN.md)为准。

## 设计目标

- 产品定位：在手机上远程操控开发者电脑中的 Claude Code、Codex 和 Cursor Agent。
- 视觉气质：安静、专注、接近高质量 IDE/终端；保持高信息密度，同时留出足够呼吸空间。
- 深色基础：使用深色背景、分层表面和细边框建立层级，不依赖明显投影。
- 强调色：赤陶色只用于主要操作、活动状态和流式光标；成功、警告、危险使用独立语义色。
- 字体：普通界面使用 UI 字体；路径、会话 ID、Git 分支、代码、Token 和命令使用等宽字体。
- 交互：卡片圆角、底部弹层和胶囊标签遵循统一半径；使用 4pt 间距网格，屏幕边距 16pt，点击区域不小于 44pt。

## 聊天页结构

- 用户消息靠右显示为气泡；Agent 回复以接近终端记录的全宽正文呈现。
- 工具调用使用可展开卡片，展示状态、工具名称、目标路径、diff 统计和命令输出。
- 输入区支持文本、图片、语音、`@` 文件/专业 Agent、斜杠命令，以及发送/停止状态。
- 模型、思考强度、执行模式和上下文用量显示在输入框上方，低频操作放入右上角快捷菜单。
- 权限请求明确展示风险、影响范围和允许/拒绝操作，高风险操作要求二次确认。
- 流式输出、后台任务、子 Agent 和活动记录均需有稳定、可追溯的状态表现。

## 相关页面

完整产品包含聊天、权限审批、会话列表、目录选择、设备配对、电脑选择和设置等页面。后续实现又扩展了项目首页、Token 用量、Codex 目标/审查、Skills/Plugins、MCP/Apps 和 Cursor 模型选择等能力。

## 归档结论

旧对话中的 HTML/JSX 草稿只是概念验证，不是生产代码，也不应作为当前 UI 的像素级依据。修改界面时应优先检查 Compose 实现、当前中文字符串和设计文档，并在真机尺寸上验证键盘、滚动、返回位置与安全区行为。
