# 版本更新记录

## 1.3.8 — 2026-07-18

### 新增

- 设置 → 关于新增当前 daemon 版本；版本由电脑端在每次加密重连时上报，服务器升级 daemon 后无需重新配对即可自动刷新。
- daemon 托管运行中提示词队列，App 可查看、立即追加或移除，断线重连后恢复真实队列状态。
- 移动端统一工作区入口，集中展示会话改动、文件/Markdown 预览、diff、最近终端记录与远程终端入口。
- 执行中的计划与子 Agent 进度聚合卡片，以及按项目保存的 Agent、模型、权限模式和推理强度偏好。
- iPhone 主屏幕 Token 用量小组件，提供小号与中号布局，显示今日 Token、请求数、Codex 每周剩余额度、套餐和快照更新时间；App 连接 daemon 或回到前台时自动刷新共享快照。
- 手机到 daemon 的常规文件与视频附件上传，支持发送前状态、上传进度、消息内文件卡片、视频预览与工作区落盘。
- Face ID / 生物识别 App 锁，支持启动和回前台自动加锁，并在进入系统任务切换器前显示隐私遮罩。
- 移动端打开任意项目目录，以及移动端、桌面端会话重命名。
- 为发送、停止生成、语音操作、设置开关以及模型、思考深度、操作模式和 Usage 时间范围等关键操作增加触觉反馈。
- MCP 服务器支持查看具体工具、资源与资源模板的名称、说明、URI 和类型；Apps 无可用集成时提供明确空状态说明。

### 调整

- 未配对用户首次打开 App 时先进入电脑端安装引导，完成安装后再进入扫码／六位配对码页面；添加另一台电脑仍直接进入配对页。
- 安装引导的命令卡、步骤编号和页脚改用主题语义色，避免深色模式出现突兀黑块、浅色模式出现黑色命令框。
- 操作模式改用与模型、思考深度一致的图标芯片、单选弹出菜单和设置页分段控件；全自动模式继续要求危险操作二次确认。
- 移除输入框旁重复的模型选择入口，模型选择统一保留在输入框上方状态栏。
- 亮色主题恢复暖色半透明材质和卡片层次，避免页面被大面积纯白模块覆盖。
- 移动端页面底布改为稳定的不透明主题色，卡片和浮层继续保留玻璃材质。

### 修复

- 修复根命令 `cc-pocket-daemon --version` 错误显示帮助页；现在无需子命令即可输出内置版本。
- 修复部分 Debian/systemd 环境升级完成后因 Java 进程未从 PATH 找到 `systemctl` 而抛异常；updater 现在优先使用绝对路径，重启失败时保留已安装版本并给出手动操作提示。
- 修复安装引导的“完整安装指南”指向未启用的 GitHub Pages 而返回 404；现在直接打开仓库内的使用文档。
- daemon 启动时将 Claude、Codex、Cursor 都作为可选后端，Codex-only 主机不再因找不到 Claude CLI 而退出；正式 `1.3.28` Release 同时提供 `SHA256SUMS`。
- 修复 MCP 与 Apps 页面因集成更新通知自触发而持续闪烁，并将 403/HTML 响应整理为可读错误信息。
- 修复 MCP 认证状态 `bearerToken` 被直接显示成疑似按钮的问题；已认证服务器现在显示“已连接”，只有未登录状态可点击授权。
- 修复 iOS 顶部状态栏和底部 Home Indicator 安全区露出不同颜色背景的问题。
- 聊天输入区使用四角一致的圆角和 6dp 四周留白，形成完整的悬浮卡片效果。
- 修复深色模式径向渐变在手机屏幕上出现一圈圈色阶波纹的问题。
- 修复浅色模式下聊天记录、工具调用和状态信息的二级文字对比度过低、难以辨认的问题。
- 修复从主页面会话列表切换时，旧会话迟到的响应覆盖新选择、导致跳回之前对话的问题。
- 升级 Codex 会话标题：对直接复制首条问题的标题自动去除口语前缀、长链接和冗余尾句，生成简短的会话摘要；手动改名仍保持最高优先级。
- 修复 TrollStore Token Widget 打码占位或黑屏：主 App 与 Widget 扩展改用包含 XML/DER entitlement 和完整标识的 ad-hoc 签名，云端打包会同时校验 App Group 与嵌套扩展签名后才上传 IPA。
- 修复 Token Widget 只识别 Codex 周额度、在账户仅返回 5 小时额度时隐藏进度条的问题；现在优先显示周额度并自动回退到 5 小时额度。
- 修复文件附件图标在 Kotlin/Native iosArm64 编译时的符号冲突。

### 兼容性

- 可管理提示词队列需要匹配的新版 App 与 daemon；旧端会忽略新增协议帧，其余会话功能保持兼容。
- 可管理提示词队列、文件/视频附件和会话重命名需要匹配的当前 App 与 daemon；daemon 热修复 Release 已更新至 **1.3.30**。
- iPhone 小组件通过 App Group 读取最近一次用量快照；受 WidgetKit 调度限制，它不会自行保持 daemon WebSocket 长连接。

### 版本

- App：**1.3.8**。
- daemon：版本上报功能需要 **1.3.31 或更高版本**。

## 1.3.7 — 2026-07-18

### 调整

- 生产 Relay 迁移到 `wss://relay.txx.app`，统一通过 HTTPS/WSS 的 443 端口提供配对与 WebSocket 服务。
- App 自动把原官方 Relay 地址迁移到新域名，同时保留设备凭据和配对关系；自托管地址不受影响。
- daemon 自动改写旧 systemd/launchd 启动参数中的官方 Relay 地址，升级二进制后无需先重装服务。
- 新增 Nginx Proxy Manager Docker 部署模板和 Relay GitHub Artifact 云端打包流程。

### 版本

- App：**1.3.7**。
- daemon / Relay artifact：**1.3.25**。

## 1.3.6 — 2026-07-16

### 新增

- Codex 原生运行中追加指令、上下文压缩、会话分支、归档/恢复、目标和代码审查。
- 在 App 内管理 Codex Skills、Marketplace Plugins、MCP 服务器和 Apps。
- Codex 账号用量只展示提供方明确返回的限额窗口；账号提供重置次数时支持执行限额重置。

### 调整

- App 全面采用跨 iOS、Android 与桌面端一致的毛玻璃材质、环境光背景和半透明层级，保留亮色、暗色与系统主题。
- 流式消息跟随由 20Hz 提升至约 30Hz；缓存玻璃背景画笔与状态列表快照，减少滚动和输入期间的无关分配与重组。
- 桌面悬停、设置分页与快捷操作使用统一的 150ms 可中断过渡，主要弹层共用同一玻璃容器。
- 项目页统一使用扁平分组列表，移除设置按钮前的列表/树状分支切换按钮和树状浏览逻辑。
- 完整移除 xAI 电话助手的 App 设置、通信协议、daemon 生命周期、命令与部署说明。
- 模型、思考强度和执行模式统一放在输入框上方，不再在右上角快捷菜单重复显示。
- 新增以 GitHub artifact 为主的 Linux daemon 升级指南，生产服务器无需从源码构建。
- 项目维护文档统一为简体中文。

### 版本

- App：**1.3.6**。
- daemon artifact：**1.3.22**。

## 1.3.5 — 2026-07-13

相较 1.3.4，本版本重点改善三个 Agent 的一致性、远程控制安全、会话连续性和移动端响应速度。

### 新增

- 完整支持 Cursor Agent 会话、原生历史发现、账号模型动态发现、模型 variant、用量统计和会话恢复。
- 权限风险分级、影响范围摘要、高风险操作二次确认和授权审计记录。
- 按时间排列审批、拒绝、工具执行、后台任务和完成事件。
- 在 Token 用量中显示 Codex 官方账号限额快照、每周剩余、重置时间、套餐、Credits 和实时刷新状态。
- 跨设备接管说明、返回电脑摘要和会话阅读位置连续性。
- 输入框 `@` 中文专业 Agent 补全，以及更清晰的空会话建议。

### 改进

- Claude、Codex、Cursor 使用统一的新建会话自主等级、卡片高度、间距、选中状态和全自动警告。
- 聊天模型和思考控制改为紧凑状态栏，在输入框上方打开单选面板。
- 输入第 6 位配对码后自动连接、收起数字键盘，并提供更清晰的相机/错误恢复提示。
- 聊天恢复正确阅读位置，键盘变化时保持底部定位，并确保审批卡片可操作。
- 长回复以受控频率刷新，减少重组负担和界面卡顿。
- 项目、离线、空状态、注意事项、导航和状态入口使用更大的点击区域与更清楚的恢复操作。
- 项目头像不再使用生成字母；Codex 使用 OpenAI 官方 Blossom 图形，iOS 图片附件使用系统 `photo` 图标。
- 桌面切换、最近会话持久化、通知、提示词投递、relay 重连和长会话回放更加稳定。

### 修复

- 最近聊天内容被键盘遮挡，或重新打开后跳到顶部。
- 重连、进程重启和回合确认期间的提示词丢失、重复回复或重复回放。
- Cursor 重复回复、模型 ID 传递、会话列表和模型选择器加载问题。
- Codex 每周额度展示和跨平台重置时间计算。

### 兼容性

- App 版本：**1.3.5**。
- 本节所列功能需要 daemon **1.3.5 或更高版本**。尚未发布部分的新 Codex 集成功能需要匹配的当前 App 与 daemon；不要为了匹配 App 营销版本而降级较新的 daemon。
