一句话说明

CC Pocket 是 Claude Code 与 OpenAI Codex 的跨平台远程客户端，不是云端 IDE，也不托管模型；代码和代理都留在你的电脑上。

CC Pocket 能做什么？

  • 从手机或另一台电脑启动、恢复并接管 Claude Code / Codex 会话。
  • 实时查看文本、代码块、工具事件、后台任务和多代理工作流。
  • 远程允许或拒绝工具权限，查看逐行 diff，并在需要时中断运行。
  • 在同一个客户端管理多台已配对电脑和多个项目。
  • 同一局域网可直连；不在局域网时使用端到端加密中继。

和官方移动方案有什么区别？

Anthropic 已提供 Claude Code Remote Control，OpenAI 也已提供 Codex 的 ChatGPT 移动体验。官方方案适合只使用对应厂商代理、希望获得第一方集成的用户。

CC Pocket 的差异是：一个客户端同时支持两种代理；协议、守护进程、中继与客户端以 MIT 许可证开放；中继可自托管；设备直接配对，不需要注册 CC Pocket 账号。厂商账号或模型使用资格仍由 Anthropic / OpenAI 决定。

安全边界

客户端与本机 daemon 之间使用 P-256 ECDH、HKDF-SHA256 与 AES-256-GCM 建立端到端加密通道。中继负责鉴权和转发密文，不持有端点私钥或会话密钥。

必须理解的限制：端到端加密不等于本机沙箱。daemon 以你的操作系统用户权限驱动代理；全自动模式可能跳过逐步批准。项目的自定义 Noise 风格通道目前也没有经过独立安全审计。
